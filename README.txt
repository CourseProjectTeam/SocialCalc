# спек

Работа пользователя
1.	логин-пароль (помимо ввода лога и пароля можно предложить демо-вход)
2.	основная страница:
пользователь видит поле предназначенное для вода чека. 
2а.Состоит из двух частей, строки (для забивки название продукта и его цены, двойное поле – наименование/цена) и столбиков (для забивки имен). 
2б.Пространство между ними представлет собой сетку типа крестика нолика в которой крестиком обозначается используется юзером(имя юзера указано в столбце) указанный продукт (указано в строчке). По умолчанию пользователь не пользуется продуктом
(пустая клетка на пересечении строки и столбцы). 
2в.Изначально на поле отображены одна строка и один столбик (новые поля добавляется за счет стрелочки с плюсика или что то подобного, располагается под первой строчкой и сбоку первого столбика. При нажатия на стрелочку добавляется столбик или строчка, стрелочка сдвигается/опускается за новосозданную строчку/стобец (создавать возможность удалить строчку/столбик, за исключением первой).
3.После нажатие кнопки далее( и проверки вводимых данных автоматически) данные отправляется на сервер где производится расчет. Пользователю возвращается страницу на которой отображается расчет. Расчет выглядит следующим образом – каждому пользователю представляется список только тех покупок которые он будет использовать ( берется из таблицы 2а), цена за них, сколько он заплатил и сколько он должен( или кто ему должен и сколько);
4. Далее пользователю предлагается сохранить чек(виде какого либо файла), выслать его на почту или создать новый.

Внутренняя работа
I.	Механизм входа
1.	открывается страница входа(1) с полями для ввода (логин(1а), пароль(1б)), кнопкой сабмита(1в) 
2.	если пароль(1б) и логин(1а) введены в правильной форме, данные отправляются на сервер в json 
3.	если сервер отвечает, что такой пользователь существует, перенаправляем на страницу профиля (2) 
если сервер отвечает, что такого пользователя нет или данные невалидны, снова открываем страницу 1

II.	Работа с сайтом
1.	видим таблицу из двух столбиков(я, поле для поиска другого пользователя) и одной строки, сбоку от которой отображается 2 поля: текстовое для наименования айтема и строковое для цены айтема. Сверху слева 2 кнопки добавления строки/столбца(2а, 2б). Сбоку слева от каждой строки кнопки для удаления строк(2д), а вверху каждого столбца кнопка для удаления столбца(2е). Внизу есть кнопка подтверждения(2з). Видим сетку с ячейкой для отметки принадлежности айтема к пользователю(2ж)
