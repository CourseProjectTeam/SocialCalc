Спек

# Работа пользователя
Страницы:
1.	логин-пароль 
2.	основная страница:
пользователь видит страницу, предназначенную для вода чека. Поле для ввода текста для поиска пользователей по сайту, чтобы прикрепить их к данному чеку. 
Снизу строчки для ввода покупок из чека (расширяемое количество строк). Минус и плюс сверху поля имени предназначены для удаления или добавления строки. 
Каждая строка состоит из:
-текстового поля для наименования
-числового поля для цены
-выпадающего списка для выбора пользователя, оплатившего покупку из тех, кто был прикреплён к чеку
Снизу кнопка для очистки формы и для отправки формы.
После отправки формы и проведения необходимых расчётов возвращается ссылка, по которой расположена страница чека с результатами расчётов(3), доступная только тем, кто прикреплён к чеку. 
3. Там отображается список покупок с ценами по каждому юзеру и долг для конкретного пользователя, который просматривает чек.
(Потом там будут кнопки для управления чеком: возможность вернуть долг или послать сообщение об отказе).
(Если успеем, сделаем демо-вход без регистрации и отправку чека на e-mail или в файл)

# Внутренняя работа

I. Механизм входа
1. открывается страница входа(1) с полями для ввода (логин, пароль), кнопкой сабмита
2. если пароль и логин введены в правильной форме, данные отправляются на сервер в json (см. json шаблоны)
3. если сервер отвечает, что такой пользователь существует, перенаправляем на страницу профиля (2) 
если сервер отвечает, что такого пользователя нет или данные невалидны, снова открываем страницу входа (1)

II. Работа с сайтом
4. В форму на основной странице(2) вводятся данные, которые отправляются на сервер в json  (см. json шаблоны)
5. данные по чеку уходят в БД, параллельно проводятся расчёты общей суммы чека, доли каждого заплатившего, долга каждого, включённого в чек
6. формируется страница с результатами расчётов и ссылка на неё, которая формируется по id чека в БД
8. ссылка перенаправляет на страницу 3, где отображаются данные, пришедшие с сервера (общая таблица с предыдущей страница без кнопок, и внизу результаты расчётов для пользователя). (TODO: подумать, как это должно выглядеть)
